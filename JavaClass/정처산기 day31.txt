Ex3102

[정리] 추상클래스 o/x
1. 추상클래스는 무조건 추상메서드를 가져야 한다. X
2. 추상메서드는 추상클래스 안에 작성할 수 있다. O 
3. 추상메서드는 일반 클래스 안에 작성할 수 있다. X
4, 추상메서드를 포함하는 클래스는 추상클래스일 수밖에 없다 O
5. 추상클래스는 객체생성 할 수 없다. O

[정리] 인터페이스 o/x
1. 인터페이스는 상수만 작성할 수 있고 변수는 작성훌 수 없다. O
2. 인터페이스는 본질상 완성된 메서드를 작성할 수 없다. O
3. 인터페이스는 추상메서드만 작성할 수 있다. O
4. 인터페이스에 내가 abstract라고 안 써도 모든 메서드는 본질상 추상 메서드이다. O
5. 인터페이스는 객체생성 불가능하다. O
6. 추상클래스는 extends 인터페이스는 implements한다 O

[참고] 인터페이스는 상속받을 수 있다. (extends 키워드)---> 클래스를 상속받는 게 아니라 인터페이스를 인터페이스가 상속받는 것.
 - 클래스는 다중상속 안되나, 인터페이스는 다중상속이 허용된다 (이해)
	Ex) interface I1{...}
	    interface I2 {...}
	    interface I3 extends I1, I2{...} 가능
 - 클래스가 인터페이스를 다중구현 하는 것은 허용된다.
	Ex) interface I1{...}
	    interface I2 {...}
	    class I3 implements I1, I2{...} 가능

인터페이스의 스태틱메서드와 디폴트메서드
[참고] 인터페이스의 스태틱메서드와 디폴트메서드
	JDK 1.8에서 새롭게 도입됨.
	1) 스태틱 메서드: 인터페이스에 기본으로 완성
			static 키워드를 사용함
	2) 디폴트메서드 : 인터페이스에 기본적으로 완성되어져 있음
		default 키워드를 사용함
		인터페이스를 구현하는 클래스에 기본적으로 존재하게 되는 완성된 메서드.
